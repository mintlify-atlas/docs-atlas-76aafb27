Team Talker is a collaborative whiteboarding tool that lets teams work together in real-time on a shared canvas. This guide will help you set up and run your first server.

<Steps>
  <Step title="Clone the repository">
    First, clone the Team Talker repository to your local machine:

    ```bash
    git clone https://github.com/Temiloluwa-Ogundiran/teamtalker.git
    cd teamtalker
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using npm:

    ```bash
    npm install
    ```

    <Note>
      This project uses Next.js, Prisma, and tldraw for the collaborative canvas experience.
    </Note>
  </Step>

  <Step title="Set up environment variables">
    Create a `.env` file in the root directory with the following variables:

    ```bash
    # Clerk Authentication
    NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key
    CLERK_SECRET_KEY=your_clerk_secret_key
    NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
    NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
    NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/
    NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/

    # Database
    DATABASE_URL=your_postgresql_connection_string

    # UploadThing for file storage
    UPLOADTHING_SECRET=your_uploadthing_secret
    UPLOADTHING_APP_ID=your_uploadthing_app_id
    ```

    <Warning>
      You'll need to set up accounts for:
      - **Clerk** for authentication: [https://clerk.com](https://clerk.com)
      - **UploadThing** for file storage: [https://uploadthing.com](https://uploadthing.com)
      - **PostgreSQL** database (local or hosted)
    </Warning>

    <Tip>
      For local development, you can use a PostgreSQL connection string like:
      `postgresql://username:password@localhost:5432/teamtalker`
    </Tip>
  </Step>

  <Step title="Initialize the database">
    Run Prisma migrations to set up your database schema:

    ```bash
    npx prisma generate
    npx prisma db push
    ```

    This creates the necessary tables for profiles, servers, and members.
  </Step>

  <Step title="Start the development server">
    Run the development server with the collaborative websocket:

    ```bash
    npm run dev
    ```

    This command starts:
    - Next.js development server on `http://localhost:3000`
    - Y-websocket server on `http://127.0.0.1:1234` for real-time collaboration

    <Note>
      The `dev` script uses `concurrently` to run both the Next.js server and the Y-websocket server simultaneously.
    </Note>
  </Step>

  <Step title="Create your first server">
    Now you're ready to create your first collaborative workspace!

    1. Open your browser and navigate to `http://localhost:3000`
    2. Sign in or create an account using Clerk authentication
    3. Click the **Create Server** button or use the initial modal
    4. Give your server a name and upload an image
    5. Click **Create** to launch your server

    You'll be redirected to your server's canvas page at `/servers/[serverId]`
  </Step>

  <Step title="Access the collaborative canvas">
    Once your server is created, you'll see the tldraw canvas interface where you can:

    <CardGroup cols={2}>
      <Card title="Draw & Collaborate" icon="pen">
        Use drawing tools to create shapes, add text, and sketch ideas in real-time
      </Card>
      <Card title="Invite Team Members" icon="users">
        Generate invite links to bring collaborators to your server
      </Card>
      <Card title="Manage Members" icon="user-gear">
        Change roles between ADMIN, MODERATOR, and GUEST
      </Card>
      <Card title="Real-time Sync" icon="arrows-rotate">
        All changes are synchronized instantly using Yjs websockets
      </Card>
    </CardGroup>

    <Tip>
      Use the color picker in the top-right corner to customize your cursor color and make it easier for team members to identify your contributions.
    </Tip>
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Websocket connection fails">
    Make sure the Y-websocket server is running on port 1234. Check that no other process is using this port:
    
    ```bash
    lsof -i :1234
    ```
  </Accordion>

  <Accordion title="Database connection errors">
    Verify your `DATABASE_URL` is correct and the PostgreSQL server is running. Test the connection:
    
    ```bash
    npx prisma studio
    ```
  </Accordion>

  <Accordion title="Clerk authentication not working">
    Double-check your Clerk environment variables are set correctly. The publishable key should start with `pk_` and the secret key with `sk_`.
  </Accordion>

  <Accordion title="File uploads failing">
    Ensure your UploadThing credentials are configured correctly. Check the UploadThing dashboard for your secret and app ID.
  </Accordion>
</AccordionGroup>
