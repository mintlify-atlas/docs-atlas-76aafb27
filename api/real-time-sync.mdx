## Overview

TeamTalker implements real-time collaborative canvas synchronization using:

- **Yjs** - CRDT-based data structure for conflict-free real-time collaboration
- **y-websocket** - WebSocket provider for Yjs synchronization
- **tldraw** - Collaborative whiteboard/canvas library
- **WebSocket server** - Running on port 1234 for real-time sync

## Architecture

The real-time sync system consists of three main components:

1. **WebSocket Server** - Broadcasts changes between clients
2. **Yjs Document** - Shared data structure with CRDT conflict resolution
3. **tldraw Store** - Canvas state and drawing operations

```
Client A ─────┐
              ├──► WebSocket Server (port 1234) ──► Yjs Document
Client B ─────┤                                            │
              │                                            ▼
Client C ─────┘                                     tldraw Canvas
```

## Document Synchronization

The `useYjsStore` hook implements bidirectional synchronization between the local tldraw store and the shared Yjs document.

### Store → Yjs Document Sync

When users make changes to the canvas:

```typescript
store.listen(
  function syncStoreChangesToYjsDoc({ changes }) {
    yDoc.transact(() => {
      Object.values(changes.added).forEach((record) => {
        yStore.set(record.id, record);
      });

      Object.values(changes.updated).forEach(([_, record]) => {
        yStore.set(record.id, record);
      });

      Object.values(changes.removed).forEach((record) => {
        yStore.delete(record.id);
      });
    });
  },
  { source: "user", scope: "document" }
);
```

Source: `/useYjsStore.ts:71`

### Yjs Document → Store Sync

When remote changes arrive:

```typescript
const handleChange = (
  changes: Map<string, { action: 'delete' | 'update' | 'add'; ... }>,
  transaction: Y.Transaction
) => {
  if (transaction.local) return;

  const toRemove: TLRecord["id"][] = [];
  const toPut: TLRecord[] = [];

  changes.forEach((change, id) => {
    switch (change.action) {
      case "add":
      case "update": {
        const record = yStore.get(id)!;
        toPut.push(record);
        break;
      }
      case "delete": {
        toRemove.push(id as TLRecord["id"]);
        break;
      }
    }
  });

  store.mergeRemoteChanges(() => {
    if (toRemove.length) store.remove(toRemove);
    if (toPut.length) store.put(toPut);
  });
};
```

### Receiving Remote Presence

```typescript
const handleUpdate = (update: {
  added: number[];
  updated: number[];
  removed: number[];
}) => {
  const states = room.awareness.getStates() as Map<
    number,
    { presence: TLInstancePresence }
  >;

  const toRemove: TLInstancePresence["id"][] = [];
  const toPut: TLInstancePresence[] = [];

  for (const clientId of update.added) {
    const state = states.get(clientId);
    if (state?.presence && state.presence.id !== presenceId) {
      toPut.push(state.presence);
    }
  }

  for (const clientId of update.updated) {
    const state = states.get(clientId);
    if (state?.presence && state.presence.id !== presenceId) {
      toPut.push(state.presence);
    }
  }

  for (const clientId of update.removed) {
    toRemove.push(
      InstancePresenceRecordType.createId(clientId.toString())
    );
  }

  store.mergeRemoteChanges(() => {
    if (toRemove.length) store.remove(toRemove);
    if (toPut.length) store.put(toPut);
  });
};
```

## Store Initialization

When a client first connects, the store is initialized with either:

1. **Existing Yjs data** (if the room has existing content)
2. **Default store** (if the room is empty)

### Loading Existing Content

```typescript
if (yStore.yarray.length) {
  // Replace the store records with the yjs doc records
  transact(() => {
    store.clear();
    const records = yStore.yarray.toJSON().map(({ val }) => val);
    store.put(records);
  });
} else {
  // Create the initial store records
  yDoc.transact(() => {
    for (const record of store.allRecords()) {
      yStore.set(record.id, record);
    }
  });
}
```

## Connection Status

The hook tracks connection status and updates the store accordingly.

### Status States

<ResponseField name="loading">
  Initial state while connecting and syncing
</ResponseField>

<ResponseField name="synced-remote (online)">
  Connected and actively syncing with the server
</ResponseField>

<ResponseField name="synced-remote (offline)">
  Disconnected but retaining local state
</ResponseField>

### Status Change Handler

```typescript
function handleStatusChange({
  status,
}: {
  status: "disconnected" | "connected";
}) {
  if (status === "disconnected") {
    setStoreWithStatus({
      store,
      status: "synced-remote",
      connectionStatus: "offline",
    });
    return;
  }

  if (status === "connected") {
    if (hasConnectedBefore) return;
    hasConnectedBefore = true;
    room.on("synced", handleSync);
  }
}
```

### Record Types

tldraw uses various record types:

- **Shape records** - Drawing shapes (rectangles, lines, etc.)
- **Instance records** - Canvas state (zoom, pan, selection)
- **Presence records** - User cursors and selections
- **Page records** - Canvas pages
- **Asset records** - Embedded images and files

## Conflict Resolution

Yjs uses CRDT (Conflict-free Replicated Data Types) to automatically resolve conflicts when multiple users edit simultaneously.

### How CRDTs Work

1. **Commutative operations** - Operations can be applied in any order
2. **Idempotent** - Applying the same operation multiple times has the same effect
3. **Associative** - Order of merging doesn't affect the result

This means TeamTalker handles concurrent edits without requiring manual conflict resolution.
